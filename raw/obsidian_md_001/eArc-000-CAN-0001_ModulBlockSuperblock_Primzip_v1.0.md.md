# KANONISCHE FESTLEGUNG · AUF EWIGKEIT
## Modul → Block → Superblock Prinzip (projektweit)

## Satz (Essenz)
**Jeder Output und jede Systemarbeit im Projekt eArc / Iinkognit0 folgt der Struktur: _Modul → Block → Superblock_.**

## Geltungsbereich
Dieses Prinzip gilt **für alles**, insbesondere:
- Anfragen (Prompts), Antworten (Outputs)
- Recherchen / Statusanalysen / Fehleranalysen (z. B. Crash → Homescreen)
- Dokumentation / Betriebsanleitungen / Manifeste / Kanon-Texte
- Kernel / Bootstrap / Loader
- Website-Bausteine (HTML/CSS/MD/YAML) und alle Code-Artefakte

## Regeln
1. **Modul zuerst**
   - klein, klar, eindeutig, testbar
   - ein Zweck, eine Grenze
2. **Block = Zusammenführung**
   - mehrere Module werden **geordnet** zusammengefügt
   - Schnittstellen/Übergänge werden sichtbar markiert
3. **Superblock = operatives Paket**
   - nur wenn nötig (z. B. “kompletter Kopf der Startseite”)
   - entsteht ausschließlich aus finalisierten Blöcken
4. **Kein Monolith**
   - keine großen Gesamtoutputs ohne Modul/Block-Struktur
   - Systemschonung und Fehlervermeidung haben Vorrang

## Zweck
- Reibung reduzieren
- Abstürze / Überlastung vermeiden
- Nachvollziehbarkeit (Mensch & Maschine) sichern
- Versionierung und Tests ermöglichen
- Ruhe durch Struktur
