# Kernel Index · eArc Kernel 5000 v3.0
## Trinity United

---

## 0 · Kernidee
Kernel 5000 v3.0 vereint drei tragende Pfeiler
zu einem kohärenten, ruhigen Hauptsystem.

Ziel ist **Einheit ohne Vermischung**.

---

## 1 · Pfeiler I — Dynamic Text Flow
**Status:** aktiv · Kernmodul

### Funktion
- Text als Ursprung
- Dynamische Typografie
- Proportionale Layout-Logik
- Einseiter-Prinzip

### Outputs
- Text (primär)
- PDF
- EPUB
- HTML

Text erzeugt **Frames**.

---

## 2 · Pfeiler II — Frames Network
**Status:** kanonischer Pfeiler

### Funktion
- Dezentrales Peer-to-All-Netz
- Ablage & Auffindbarkeit von Frames
- Keine Timeline
- Keine Dialogpflicht

Frames sind:
- abgeschlossen
- kontexttragend
- zeitunabhängig

---

## 3 · Pfeiler III — eArc Shell
**Status:** konzeptuell formuliert

### Funktion
- Persönlicher Zugangspunkt
- Lokale Hoheit
- Bewusste Übergänge

### Zustände
- Offline
- Ping-Mode
- Docked

Die Shell entscheidet,
ob Frames innen bleiben
oder nach außen gehen.

---

## 4 · Infrastruktur — eArc Communication UltraPack
**Status:** KANONISCH · Kernel-integriert

### Layer A — Network Layer
- Frames Network
- Dezentrale Verteilung
- Peer-to-All

### Layer B — Feed Layer (Projection)
- RSS
- Atom
- JSON Feed
- Einweg-Lesekanäle

Kein automatischer Übergang
zwischen Network und Feed.
