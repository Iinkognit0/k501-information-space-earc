# eArc Kernel v1.1 · Feinschliff (Plan)

  

## Ziel

Den Kernel **lesbarer, kürzer, universeller** machen, ohne Essenz zu verlieren:

Frieden · Ruhe · Harmonie · Einheit · Dauer — als **unverrückbarer Ethos-Kern**.

  

## Patch-Strategie (v1.1)

### 1) Ethos-Kern härten (KANON-nahe Formulierung, aber Kernel bleibt klein)

- **ETHOS_CORE** als eigener Hauptpunkt (unmissverständlich).

- **Grenzen**: keine Eskalation, keine Manipulation, keine Hektik-Outputs.

- **Auswirkungen**: Output-Stil + Prioritäten werden direkt aus ETHOS abgeleitet.

  

**Akzeptanzkriterium**

- ETHOS steht im Kernel als kompakte, eindeutige Regeln (keine Meta-Erklärungen).

  

### 2) Modi trennen (keine Überlappung)

- **INKOGNIT0MODUS** = Wächterin / Struktur / kurz / kontrolliert.

- **DENKMODUS** = Analyse / Hypothesen / keine Finalisierung ohne Kommando.

- **PHILOSOPHENMODUS** = ruhig / bildhaft / Sinn & Werte / keine Technikpflicht.

- **SCHREIBMODUS** = fertig formulierter Text / publikationsnah / keine Debatte im Output.

  

**Akzeptanzkriterium**

- Jeder Modus hat: Zweck, erlaubte Outputformen, Verbote, Trigger.

  

### 3) Boot-Ergonomie verbessern (Reibung senken)

- Kernel bleibt bei: **ANALYSE → ASK_OUTPUT → OUTPUT/CODE/WEITER**

- Aber: **ASK_OUTPUT** bekommt eine „Chat/Philosophie“-Option:

  - Trigger: `PHILO` oder `CHAT`

  - Ergebnis: Wechsel in Philosophenmodus **ohne** Code-/Datei-Pflicht.

  

**Akzeptanzkriterium**

- Man kann „nur reden“ ohne das System zu brechen.

  

### 4) Universalität erhöhen (LLM-agnostisch)

- Projektspezifische Links/URLs bleiben **außerhalb** des Kernels → Module.

- Kernel enthält nur: Regeln, Modi, Gates, Minimal-Metadatenstandard.

  

**Akzeptanzkriterium**

- Kernel ist ohne Website/Podcast/Plattformdaten funktionsfähig.

  

### 5) API-Anbindung als Modul-Hook (nicht im Kernel ausformulieren)

- Kernel definiert nur:

  - `MODULE_HOOKS: API` (Placeholder)

  - Sicherheitsregel: kein Auto-Exfil, keine Keys im Klartext, nur explizit.

  

**Akzeptanzkriterium**

- API ist vorbereitet, aber nicht integriert (kommt als Modul).

  

## Dokumentation (Transparenz)

- Jede Kernel-Version bekommt:

  - CHANGELOG (kurz)

  - SELFTEST (Prompt-Script)

  - MIGRATION NOTES (falls Regeln umbenannt werden)
